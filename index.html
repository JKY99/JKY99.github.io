JPA Auto-flush 지연 업데이트 트러블슈팅

QueryDSL 조회 메서드에서 간헐적 런타임 예외가 발생하는 원인이 JPA Write-behind 메커니즘 때문임을 규명. 엔티티 수정(setter) 시점에는 SQL이 발행되지 않고 후속 QueryDSL 조회 시점에 Auto-flush가 발생하면서 지연된 UPDATE SQL 오류가 드러나는 구조였습니다. 분석 결과를 팀 디버깅 가이드로 공유했습니다.

QueryDSL 서브쿼리 기반 조인 유틸리티 설계

부모-자식 조인 시 중복 자식 문제 반복 발생. ON절 BooleanBuilder와 MAX/MIN 서브쿼리를 활용한 범용 유틸리티 메서드를 설계하여 팀 전체에서 재사용 가능하게 구현했습니다.

  • 수출센터 수기 프로세스 디지털화 (테이블 7개 신규 설계)
  • 대시보드 API 응답 시간 87% 개선 (3초 → 300~400ms)
  • JPA Auto-flush 트러블슈팅 가이드 팀 공유
  • SMS 장애 유형별 분석 체계 구축 (응답 코드 Enum 표준화)
